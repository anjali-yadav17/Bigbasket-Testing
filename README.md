# E-Commerce Platform End-to-End Testing (BigBasket)

## 📌 Project Overview
This repository contains a comprehensive software testing portfolio project based on the **BigBasket** e-commerce platform. It demonstrates a complete testing approach covering Functional, UI/UX, Performance, Security, API, and Automation testing methodologies.

**Target Website:** [https://www.bigbasket.com/](https://www.bigbasket.com/)

## 🚀 Objective
To perform end-to-end testing of an e-commerce platform similar to BigBasket to ensure quality, reliability, and excellent user experience across various devices and browsers.

## 🛠️ Tools & Technologies Used
| Testing Type | Tools Used |
| :--- | :--- |
| **Test Management** | Jira, Excel |
| **Automation Testing** | Selenium WebDriver (Java), TestNG |
| **API Testing** | Postman |
| **Performance Testing** | Apache JMeter |
| **Security Testing** | OWASP ZAP |
| **Database Testing** | SQL |

## 📂 Repository Structure
* **`Test_Plan.md`**: Overall testing strategy and scope.
* **`Test_Cases.md`**: Detailed step-by-step test scenarios for major modules.
* **`Bug_Report.md`**: Sample defect reports with reproducible steps.
* **`RTM.md`**: Requirement Traceability Matrix ensuring coverage.
* **`Test_Summary_Report.md`**: Final results and test execution metrics.
* **`scripts/`**: Contains sample automation scripts for Selenium, JMeter, and API collections.

## 🎯 Modules Tested
1. **User Authentication** (Login / Signup)
2. **Product Discovery** (Search, Categories, Filters)
3. **Shopping Cart** (Add/Remove, Quantity adjustment)
4. **Checkout & Payments** (UPI, Cards, Failed transactions)
5. **Order Management** (Placement, Tracking, History)

## 💡 Key Highlights
- Designed comprehensive test artifacts following SDLC & STLC best practices.
- Implemented automated UI tests using Selenium and Java.
- Conducted REST API validation for backend endpoints.
- Simulated high traffic using JMeter for performance assessment.
